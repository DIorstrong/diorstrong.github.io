# Decap CMS 在线编辑器 —— 使用指南

本指南将带你完成 Decap CMS 在线编辑器的配置，让你可以通过网页后台发文章、上传配图、管理音乐页面，所有更改自动提交到 GitHub 并自动部署。

---

## 使用方式（仅需 3 步）

博客已集成 Decap CMS 在线编辑器，使用 **GitHub Personal Access Token** 方式登录，无需配置复杂的 OAuth App。

### 第一步：生成 GitHub Token

1. 点击下面链接，或直接访问：
   ```
   https://github.com/settings/tokens/new
   ```
2. 填写以下信息：
   - **Note**：`博客 CMS`
   - **Expiration**：选择 `No expiration`（永不过期），或根据喜好选择有效期
   - **Select scopes**：勾选 **`repo`**（完整控制私有和公开仓库）
3. 滚动到页面底部，点击 **Generate token**
4. **立即复制生成的 token**（格式类似 `ghp_xxxxxxxxxxxxxxxxxxxx`），页面刷新后将无法再次查看

> ⚠️ **重要**：这个 token 相当于你的 GitHub 账号密码，请勿泄露给他人。

### 第二步：登录 CMS 后台

1. 打开博客管理后台：
   ```
   https://diorstrong.github.io/admin/
   ```
2. 将刚才复制的 token 粘贴到输入框
3. 点击 **登录**
4. 系统会自动验证 token 并加载 CMS 界面

### 第三步：开始使用

登录成功后，左侧边栏会显示以下内容管理板块：

| 板块 | 功能说明 |
|------|---------|
| **文章** | 发布新文章、编辑已有文章、设置封面图、标签、草稿状态 |
| **独立页面** | 创建新的独立页面（如"关于我"、"项目展示"等） |
| **音乐页面** | 直接在线编辑"音乐"页面的内容 |
| **教育经历页面** | 直接在线编辑"教育经历"页面的内容 |

## 发布一篇新文章

1. 点击左侧 **"文章"** → 右上角 **"新建文章"**
2. 填写表单：
   - **标题**：文章标题
   - **发布日期**：默认当前时间
   - **草稿**：勾选表示存草稿（不会发布），取消勾选表示正式发布
   - **标签**：可以添加多个，如 `随笔`、`生活`、`学习`
   - **摘要**：简短描述，会显示在首页卡片上
   - **封面图**：点击上传或拖拽图片，会显示在卡片顶部
   - **正文**：支持 Markdown，可以使用编辑器工具栏排版
3. 写完后点击右上角 **"保存"**，选择 **"发布"**（Publish now）
4. 内容会自动提交到 GitHub 仓库，并触发自动部署
5. 约 1-2 分钟后刷新博客首页，即可看到新文章

## 上传配图

1. 在文章编辑器的正文区域，点击工具栏的图片按钮 📷
2. 选择 **"上传"** 或直接拖拽图片到编辑器
3. 图片会自动保存到 `static/images/uploads/` 目录
4. 发布后图片会正常显示在文章中

## 编辑音乐/教育经历页面

1. 点击左侧 **"音乐页面"** 或 **"教育经历页面"**
2. 直接编辑内容（支持 Markdown 富文本编辑器）
3. 点击右上角 **"保存"** → **"发布"**

## 常见问题

### Q1：token 登录时提示"无效"？

- 检查 token 是否复制完整（包括 `ghp_` 前缀）
- 确认生成 token 时勾选了 `repo` 权限
- 如果 token 已过期，需要重新生成

### Q2：保存文章后网站没有更新？

- 保存只是提交到 GitHub 仓库，需要等待 GitHub Actions 构建完成（约 1-2 分钟）
- 可以在仓库的 **Actions** 标签页查看构建进度
- 如果构建失败，查看 Actions 日志排查问题

### Q3：如何添加新的导航菜单？

目前导航菜单通过修改 `hugo.toml` 文件配置。如果需要修改，可以：
1. 直接在 GitHub 仓库中编辑 `hugo.toml`
2. 或告诉我，我帮你修改

### Q4：退出登录后如何重新登录？

点击页面顶部的 **"退出登录"** 按钮即可。重新登录时再次输入 token。

```
你（浏览器）
  │

  │
  └──→ Decap CMS 通过 GitHub API 直接读写仓库文件

```
